SP3

A Simple, Practical, and Safe Packet Spoofing Protocol

SP3 provides a mechanism through which a server which has the capability to spoof packets can offer that capability in a limited capacity. In particular, the protocol supports spoofing packets as long as the destination client consents in advance to receive those communications.

Learn

The SP3 code lives on github. In particular, the readme explains the design of the server, and protocol.md explains the wire protocol used to send your own packets.

Demo

This demo allows you to send packets to yourself from any spoofed source. It triggers our referece SP3 server to send you a simple UDP message, which you can see if you have a publically facing machine.

Note: The basic demonstration of this capability does not handle NAT translations, and will only work if your machine has a publically accessible IP address.

To see the packet, choose a port and listen for incoming traffic, using e.g. nc -u -l 8080 . The packet below is being sent to the remote IP we detected when you visited this page. You can change it though as long as the destination also has a connection open to our server - for intricacies of the protcol, look at the protocol.md link above. When you send the packet, you will hopefully see the payload output in your netcat listener.
